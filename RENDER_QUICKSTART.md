# Render Deployment - Quick Reference

## 🚀 Quickstart - 5 Steps to Production

### 1. Deploy Eureka Server First
```yaml
Services → eureka-server
Environment Variables:
  SPRING_PROFILES_ACTIVE = prod
  EUREKA_HOSTNAME = eureka-server-YOUR-ID.onrender.com (copy from URL)
  PORT = 8761
  JAVA_TOOL_OPTIONS = -Xmx512m -Xss512k -XX:+UseG1GC
```

Get Eureka URL: After deploy, copy from Render Dashboard

### 2. Deploy Microservices
```yaml
For EACH service (user, skill-assessment, internship, recommendation, notification):
  SPRING_PROFILES_ACTIVE = prod
  EUREKA_SERVER_URL = https://eureka-server-YOUR-ID.onrender.com/eureka/
  DATABASE_URL = your-database-url
  DATABASE_USERNAME = your-username
  DATABASE_PASSWORD = your-password (use Render secrets!)
  PORT = service-specific-port
  DB_POOL_SIZE = 10
```

### 3. Deploy API Gateway
```yaml
  SPRING_PROFILES_ACTIVE = prod
  EUREKA_SERVER_URL = https://eureka-server-YOUR-ID.onrender.com/eureka/
  EUREKA_ENABLED = true
  PORT = 8080
  JAVA_TOOL_OPTIONS = -Xmx512m
```

### 4. Verify
```bash
# Check Eureka
curl https://eureka-server-YOUR-ID.onrender.com/actuator/health

# Check API Gateway
curl https://api-gateway-YOUR-ID.onrender.com/actuator/health

# Check Services Registered
curl https://eureka-server-YOUR-ID.onrender.com/eureka/apps
```

### 5. Test APIs
```bash
curl https://api-gateway-YOUR-ID.onrender.com/api/users/health
curl https://api-gateway-YOUR-ID.onrender.com/api/assessments/health
```

```

PORT=8761
JAVA_TOOL_OPTIONS=-Xmx512m -Xss512k -XX:+UseG1GC
```

```

PORT=8081                          # Change per service
EUREKA_ENABLED=true
EUREKA_SERVER_URL=https://eureka-server-YOUR-ID.onrender.com/eureka/
DATABASE_URL=jdbc:mysql://your-host:3306/db-name?useSSL=true&serverTimezone=UTC
```

## 🐛 Troubleshooting

### Error: "Cannot execute request on any known server"
**Cause**: API Gateway can't reach Eureka  
**Fix**: 
1. Check `EUREKA_SERVER_URL` is correct (should start with `https://`, not `http://`)
2. Wait 60 seconds for service registration
3. Check Eureka health endpoint

### Error: "Connection refused"
**Cause**: Database connection failed  
**Fix**:
1. Verify DATABASE_URL format
2. Add `useSSL=true` to URL
3. Check credentials are correct
4. Ensure database is accessible from internet

### Services not appearing in Eureka
**Cause**: Registration delayed or failed  
**Fix**:
1. Wait 1-2 minutes (registration takes time)
2. Check service logs for "registered" message
3. Verify `EUREKA_ENABLED=true`
4. Verify `EUREKA_SERVER_URL` is correct

### API Gateway returns 503
**Cause**: Circuit breaker opened (service down)  
**Fix**:
1. Check the downstream service's logs
2. Verify it's registered in Eureka
3. Check its database connection if applicable

### Port Already in Use
**Cause**: PORT environment variable missing or conflicting  
**Fix**:
1. Verify PORT env var is set in Render
2. Ensure each service has unique PORT
3. Render assigns unique ports automatically if not set

## 💡 Pro Tips

1. **Use Render's Built-in Logs**: No external logging needed initially
   - Services → Logs tab shows real-time output
   - Very helpful for debugging

2. **Health Checks Save Your Services**: Render uses them to detect failures
   - `healthCheckPath: /actuator/health` in render.yaml is crucial
   - Automatically restarts failed services

3. **Free Tier Memory**: 512MB is tight for Java
   - `-Xmx512m` allocates 512MB to Java heap
   - Monitor in Render logs for OOM errors

4. **Database Connection Pooling**: Limited free connections
   - `maximum-pool-size: 10` prevents exhaustion
   - Don't use defaults (20-50)

5. **Eureka Registration Time**: Not instant
   - Services take 30-60 seconds to appear
   - Don't panic if not immediate
